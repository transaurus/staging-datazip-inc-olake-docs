# OLake (v0.4.0 - v0.4.2)
February 21, 2026 – March 5, 2026

## 🎯 What's New

### Sources

1. **Column selection -** <br/> Added column-level selection for streams to allow users to select specific columns to sync, with a `sync_new_columns` parameter to control whether newly added source columns are auto-included.

## 🔧 Bug Fixes & Stability

1. **MySQL UNSIGNED integer overflow fix -** <br/> Fixed CDC values for large MySQL UNSIGNED columns by reinterpreting signed integers from the binlog parser as the correct unsigned type before processing, preventing overflow/negative outputs.

2. **Chunk-scan query performance improvement -** <br/> Replaced tuple-based range predicates with explicit lexicographic predicates and moved the logic into a shared builder, improving MySQL chunk-scan plans and performance.
​
3. **MSSQL uniqueidentifier chunk + CDC capture-instance stability -** <br/> Normalized `uniqueidentifier` (GUID) byte values for consistent chunk split/scan boundaries, and added optional `manage_capture_instances` to manage CDC capture instances during schema evolution.
